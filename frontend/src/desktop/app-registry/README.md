# Desktop App Registry Contracts

## Ownership

- `products/*/product.json` owns Product identity, UI contract, component entry, and legacy aliases.
- `frontend/src/product-runtime/*.generated.ts` is generated by `frontend/scripts/scan-products.js`.
- `component-key-map.generated.ts` and `module-icon-assets.generated.ts` are generated by `frontend/scripts/scan-modules.js`.
- Generated files are read-only outputs. Do not add manual entries to them.
- `component-key-map.ts` is the only composition point for platform/module/product component loaders.

## Registration boundaries

- A Product `productId` is the canonical app key.
- `legacyAppKeys` are compatibility aliases only and must not collide with another Product id or alias.
- A Product `entryComponentKey` must resolve to one physical Vue entry and cannot be claimed by another Product.
- Platform entries use `apps/<name>/index.vue`; module entries use `<module>/<entry>.vue`; Product-owned entries use `products/<product>/...`.
- `scan-products.js` is the build-time gate for these rules. A passing scan is required before generated registry output is trusted.
