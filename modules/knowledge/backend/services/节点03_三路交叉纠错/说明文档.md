# 节点③ 三路交叉纠错 + 回写原文

## 第一性原理

文本层的字 100% 正确，是尺子。OCR/VLM 认错的字用文本层纠回来，纠完回写 `kb_raw_data.content`，保原文干净。文本层最容易出的问题只是语序乱，不是字错。拿不准就留，别瞎纠。

## 目录

```
节点03_三路交叉纠错/
├── 对外接口.py      ★ 唯一对外暴露
├── 分支判定.py

├── 三路对齐.py

```

## 对外接口

```python
from modules.knowledge.backend.services.节点03_三路交叉纠错 import 纠错回写
# 或
from modules.knowledge.backend.services.节点03_三路交叉纠错.对外接口 import 纠错回写

result = await 纠错回写(db, document_id, owner_id)
# result: {纠正数, 回写数, 留言数, 页数, 跳过页数, 失败页数, 幂等跳过, 分支统计, 明细}
```

可选参数：

| 参数 | 默认 | 含义 |
|------|------|------|
| `启用模型裁定` | False | True 时灰区调实体裁定循环(烧 LLM) |
| `提交` | True | 回写后是否 `db.commit()` |

## 分支

| 条件 | 分支 |
|------|------|
| 有可用文本层 + 有 OCR/VLM | 三路互校(文本层当尺子) |
| 无文本层 + 有 OCR/VLM | OCR↔VLM 互校 |
| 都空/伪文本 | 跳过 |

伪文本：以 `本地图片分析` 开头的 round1 不算尺子。

## 回写规则

- 只 `UPDATE kb_raw_data.content` + `metadata_json`
- `metadata_json.纠错前原文`：备份
- `metadata_json.纠错留痕`：时间/改动明细/分支/路
- 不新增列、不删行
- 幂等：已有纠错留痕则跳过

## 复用

| 能力 | 来源 |
|------|------|
| 滑窗/CJK 窗口/护栏理念 | `semantic_align_service.canonicalize_name` |
| 灰区裁定 | `实体裁定循环.py` + `实体裁定取证工具组.py` |
| 留言表 | `kb_entity_verdict_review` |

尺子差异：实体名打齐用全库 `base_parse`；本节点用**同文档同页** text content。

## 接线

`pipeline_stages/graph.py`：实体抽取后、`align_document_entities` 前调用 `纠错回写()`。失败 try 包，不拖垮 graph。

## 查询验收

```sql
-- 看某文档是否已纠
SELECT id, page, round, source_type,
       left(content, 80),
       metadata_json->'纠错留痕' AS 留痕,
       left(metadata_json->>'纠错前原文', 80) AS 备份头
FROM kb_raw_data
WHERE document_id = :d AND owner_id = 4
  AND metadata_json ? '纠错留痕';
```
