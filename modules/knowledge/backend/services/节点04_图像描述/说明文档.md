# 节点④ 图像描述 / OCR 分家

## 职责

把「认字」和「看图」拆成干净子模块，单接口对外：

- **OCR（round2）**：只提取可见文字
- **Vision（round3）**：只描述视觉（产品/场景/版式/颜色），不重复认字

## 目录

```
节点04_图像描述/
├── 对外接口.py   ★ 描述图像(db, document_id, owner_id, page)
├── 页面渲染.py   复用 load_page_asset_bytes
├── VLM认字.py    grok-4.5-vision + knowledge_raw_ocr
├── VLM描述.py    grok-4.5-vision + knowledge_raw_vision
└── 说明文档.md
```

## 调用

```python
from modules.knowledge.backend.services.节点04_图像描述 import 描述图像

r = await 描述图像(db, document_id, owner_id, page=1)
# force=True 强制重跑已 done 的 round2/3
r = await 描述图像(db, document_id, owner_id, page=1, force=True)
```

## 模型（写死）

- profile：`grok-4.5-vision`（provider `grok-local-gateway` @ 8317）
- 入口：`describe_image_detailed(img_bytes, prompt=..., mime_type=..., profile_key=...)`
- 图：页面资产已预处理（最长边 1600 / ≤1MB JPEG，由 page_render 阶段完成）

## 与现有链关系

- **不改** `raw_collection_service.py`
- **不改** `pipeline_stages/raw_ocr.py` / `raw_vision.py`
- 本节点是补充包装；DAG 仍走原 collect 链，未来替换另信

## 幂等

`force=False` 且该页 round2+round3 均为 `status=done` 且 content 非空 → `already_done` 跳过。
