# 节点⑥ 单页融合（文本层优先 · LLM 降级）

## 第一性原理

round1 文本层 100% 正确，是尺子。有文本层时 **绝不让 LLM 改写** fused_text；LLM 最多做「补充新信息」。无文本层（图片）才走 OCR+VLM 的 LLM 兜底。

## 目录

```
节点06_单页融合/
├── 对外接口.py    ★ fuse_document / 融合文档
├── 文本层优先.py
├── LLM补充.py
├── LLM兜底.py
└── 说明文档.md
```

## 对外接口

```python
from modules.knowledge.backend.services.节点06_单页融合 import fuse_document

result = await fuse_document(db, document_id, owner_id, force=False, 启用LLM补充=False)
```

| 参数 | 默认 | 含义 |
|------|------|------|
| force | False | True 时重跑已 done 页 |
| 启用LLM补充 | False | True 时文本主导页可调 grok 追加新信息；失败 graceful |

## 分支

| 条件 | 策略 | LLM |
|------|------|-----|
| 非图片 + round1 非伪文本 + len>50 | text_dominant | 默认不调；可选补充 |
| 图片 / round1 空或伪文本 | llm_fallback | grok-4.5 |

## 模型

- profile_key：`grok-4.5`（provider=`grok-local-gateway`，本地 8317）
- 投件信写的 `grok-local-gateway` 是 provider 名；gateway chat 用 profile 名 `grok-4.5`

## 接线

仅改 `pipeline_stages/fusion.py` → `fuse_document`。  
**不删** `fusion_service.py`（索引/冲突/状态分类仍复用）。

## 幂等

`force=False` 且 `fusion_status=done` 且 `fused_text!=''` → 跳过该页。
