# 节点07A 主体枚举

## 目标
把“抽实体”从页级 LLM 改成：
1. 本地分词穷举
2. 词/组合/出现位置落盘复用
3. 语义不清的句子进试卷表，供后续批量定类

**本节点不烧 LLM。**

## 词典来源（禁止业务词硬编码）
分词用户词典与类型提示从数据库动态加载：
1. `kb_entity_dictionary`（name/category）
2. `kb_terms`（term/term_type）
3. `kb_subject_tokens`（本节点已沉淀，含 semantic_role/graph_include/hit_count）
4. `kb_subject_combos`（邻接共现，用于切法打分）

按 `owner_id` 缓存，默认 300 秒刷新。

## 动态分词（不是千篇一律）
对每一句：
1. 生成多种切法候选  
   - 词典最长优先  
   - **强词强制拆**（抑制历史误粘长串）  
   - jieba  
   - 原子切 + 词典合并  
   - jieba 后再合并  
2. 结合历史信号打分（平均质量，不是碎片总分）  
   - 词频 / 文档频 / 来源权重  
   - 历史类型与语义角色  
   - 邻接共现  
   - **覆盖率硬约束**（丢词重罚）  
   - 完整公司/产品整词奖励  
   - 可拆已知主体的粘连串惩罚  
3. 选最优切法落盘

词典权重：
- `kb_entity_dictionary` / `kb_subject_tokens`：高  
- 无类型 `kb_terms`：低，且限制吸收，避免 70 万碎片带偏

## 唯一出口
- `枚举文档(db, document_id, owner_id)`
- `枚举(db, owner_id, document_ids=None, limit=50)`

## 落盘表
- `kb_subject_tokens` 词库存
- `kb_subject_occurrences` 出现位置
- `kb_subject_combos` 邻接组合
- `kb_subject_exam_items` 待批量定类试卷

## 语义原则（重要）
1. **每个词组都要有语义角色** `semantic_role`  
   例如：业务类型:品牌 / 修饰限定 / 计量数值 / 模板套话 / 功能连接 / 待定主体
2. **`graph_include` 只表示是否默认进入主知识图谱/主召回**  
   `false` ≠ 无语义，更 ≠ 可删除
3. 旧标签“噪音”应理解为：  
   **有语义，但不作为主图谱主体**；仍参与共现、上下文完整性和后续算法召回
4. 试卷题是给“待定语义”补标注，不是把词扔掉
