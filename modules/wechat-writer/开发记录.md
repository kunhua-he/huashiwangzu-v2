# 公众号写作助手 开发记录

## 做什么

实现"选题建议 → 大纲 → 成文初稿 → 配图/素材建议 → 草稿管理"一条龙的公众号写作辅助模块。嵌入假桌面，供华哥和小马仔使用。

## 关键决策

1. **提示词存 DB 而非硬编码**：参考 agent 模块的 prompts 做法，所有人设/调性/各步骤提示词都存储在 `wechat_prompts` 表。用户可通过前端实时修改，华哥后续调整不用改代码。

2. **写作模型选 deepseek-v4-flash**：fast 模型，公号写作对延迟敏感。如需更高质的初稿可切 `deepseek-v4-pro`。

3. **成分校验走双轨制**：先调知识库 `knowledge:search` 搜索已有知识，再让 AI 做成分科学审核。知识库有内容时给出引用，没内容时 AI 审核兜底。

4. **`onupdate` 手动管理**：`update_draft` 里显式设置 `updated_at`，避免 SQLAlchemy ORM 的 `onupdate` 在 async session 下不可靠。

5. **不依赖前端 element-plus 包**：前端组件用原生 HTML/CSS 实现，避免 Rolldown 对 `element-plus/es` 的解析问题（im 模块有相同问题）。

## 踩的坑

1. **`from app.database import Base` 不存在**：`Base` 在 `app.models.base` 中，非 `app.database`。启动时报 `cannot import name 'Base'` 后改正。

2. **前端 `platform.call` 应为 `platform.modules.call`**：runtime 的 `modules` 命名空间下才有 `call` 方法。

3. **DB 建表用 `Base.metadata.create_all` + `engine.begin()`：`WechatDraft.__table__.create(engine)` 在 async 下可能出问题（跟其他模块的 pattern 一致，用 `Base.metadata.create_all` + async engine）。

4. **`onupdate=datetime.now` vs `text("CURRENT_TIMESTAMP")`**：rule 说整数列 server_default 用 text() 包起来，datetime 列也有类似规则。最终统一用 `text("CURRENT_TIMESTAMP")`。

## 待深化点

1. **配图/素材推荐**：MVP 未接素材库推荐配图，建议后续从素材库（文件系统）按关键字匹配图片。

2. **文章排版**：当前只出纯文本初稿，公众号排版（标题样式/引用/分隔线/封面图）需后续做成可视化编辑器。

3. **知识库内容建设**：成分功效校验当前走 AI 兜底，知识库搜索返回 0 条（知识库尚无护肤相关文件）。建议把俏小喵产品成分表/行业论文/历史文章入库。

4. **历史爆款学习**：如果有历史阅读量高的文章，喂给 AI 做风格参考效果更好。

5. **异步生成**：长文章生成（>3000 字）可走 `tasks.submit` 不阻塞前端。

6. **多版本管理**：当前简单 version +1 机制，后续可做 diff 对比和版本回退。

7. **发布集成**：MVP 只出草稿，后续建议对接公众号 API 实现一键发布。

8. **写作模型调参**：`deepseek-v4-flash` 生成文章时温度/top_p 未调优，可根据初稿质量微调。

9. **前端构建**：当前 `im` 模块有 `element-plus/es` 的 Rolldown 解析问题导致整体构建失败。`vue-tsc` 类型检查已通过，模块在 sandbox 中可独立运行。
