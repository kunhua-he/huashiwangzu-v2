## 任务目标

本信只解决一个节点：把 `knowledge:search` 从“返回文本块的检索”升级成“返回证据包的证据控制面”。
不要顺手做全量 workflow 平台，不要顺手做多代理理解环（信6）。

## 为什么现在做这个

- 当前 knowledge 已经有检索、融合、图谱、治理候选，但 Agent 侧仍主要把它当作“返回文本块的 search”。
- 如果不把它升级成证据控制面，后续理解环、多模型、review fork 拿到的还是噪音材料，不是结构化证据。
- 这是 A5（模型治理）和 A6（理解环）的**材料前置**。
- 检索不是终点，真正值钱的是“可回答性判断 + 证据链组织 + 任务态召回”。

## 当前事实（V2）

- 文件：
  - `modules/knowledge/backend/router.py:428-435` `api_search()` 直接调 `hybrid_search()`
  - `modules/knowledge/backend/router.py:589-719` `_cap_search()` 返回 enriched results，补 `page_fusion` 和 `document_name`
  - `modules/knowledge/backend/services/search_service.py:29-232` `hybrid_search()` 只做 keyword + vector + RRF + optional rerank
  - `modules/knowledge/backend/services/fusion_service.py:172-205`、`284-337` 已持有 `page_summary / page_title / evidence_json / confidence`
  - `modules/knowledge/backend/services/entity_service.py:546-637` 图谱与证据抽取
- 当前行为：调一次 capability → 拿到带页级融合的结果列表
- 当前缺口：
  - 没有 query rewrite
  - 没有 answerability judgment
  - 没有 evidence plan
  - 返回的是结果列表，不是结构化证据包

## 参考事实（外部更强实现）

  - 强在 dataset / pipeline / observability，检索是 workflow 的一个节点

- 它们强在：“有没有答案”和“证据够不够”是显式判断，不是让主模型自己猜。

## 本次要改什么

### 1. 新增 3 个 capability
- `knowledge:plan_evidence` —— 给定问题，规划要找哪些证据
- `knowledge:judge_answerability` —— 判断现有证据能否回答（answerable / weak / insufficient）
- `knowledge:get_citation_package` —— 返回结构化引用包

### 2. 升级 `knowledge:search`
- 从结果列表升级成 evidence packet 友好结构
- **保留兼容**：现有 `knowledge:search` 调用方不立即全部打断

### 3. search service 新增中间层
在 `search_service.py:29-232` 检索逻辑前后加：
- query rewrite（可先规则化）
- answerability judgment（可先启发式）
- 检索逻辑接受 plan 输入，而不是只吃原 query

### 4. fusion 输出扩成证据包
- `fusion_service.py:284-337` 输出从“融合文本”扩成“证据包”

### 5. 新增 evidence schema
```
claim          —— 这条证据支持的论点
source_file_id —— 来源文件
chunk_id       —— 来源分块
page           —— 页码
confidence     —— 置信度
support_type   —— 支持类型（直接/间接/反例）
```

## 本次不要改什么

- 不要重做全量知识库 UI
- 不要顺手重构所有 parser（那是信11）
- 不要把 workflow 平台一口气拉进来
- 不要改动无关模块边界
- 不要改 runtime 主循环（信1 已定型）

## 精度控制手柄

至少把以下手柄做成显式参数或显式判断：
- query 是否 rewrite
- 是否允许 multi-hop
- 是否必须生成 citation package
- evidence strength 不足时是否禁止直接回答
- answerability 是否返回 `answerable / weak / insufficient`

## 验收

### 单测
- 新增 `backend/tests/test_knowledge_evidence.py`：plan_evidence / judge_answerability / citation package
- 现有 `/api/knowledge/search`、`knowledge:get_page_fusion`、`knowledge:get_graph_context` 不退化

### 活系统
- `call_capability knowledge:search` 返回 evidence-friendly 结构
- 新增 3 个 capability 能被注册表发现（`capabilities` 工具能看到）
- 造一个“证据不足”的问题，确认返回 `insufficient` 而不是硬给文本块

### 行为口径（做完必须全部成立）
1. `knowledge:search` 返回 evidence packet（含 citation/source/page）
2. 证据不足时能显式区分“不够回答”，而不是硬给文本块
3. citation 中至少能落到 `document_id/page/chunk/source`
4. Agent 侧能消费 citation / source / page
5. 现有调用方不被立即打断（兼容）
