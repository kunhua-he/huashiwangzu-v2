## 任务目标

本信只解决一个节点：把分散在多处的模型治理逻辑**收口成 ModelRouter**，引入 provider/capability/variant/policy 四层概念，并给 4 层叠加的 retry/fallback 加上**统一重试预算**。
不要顺手做前端管理台，不要顺手做全量多代理角色切换（信6）。

## 为什么现在做这个

- 当前 V2 已有 gateway、retry、fallback、usage logging，但本质仍是静态 provider/profile/fallback。
- **审计点名（§2.4 + §1③）**：模型调用有 **4 层嵌套的 retry/fallback**，各层彼此不感知对方的重试状态，一次调用理论最多跑 `3×N×1 = 3N` 次实际 LLM 请求，但没有任何统一预算控制。这是 bug 温床。
- 如果不先把模型治理对象化，后续理解环、review、planner、retrieval 都只能挤在一个模型或线性 fallback 上。
- 这是后续“按阶段选模型、按风险切模型、按预算降模”的前置。

## 本次要改什么

### 1. 配置层引入 4 层概念
`models.json` 从静态配置升级为：
- `provider`
- `model capability`（模型能力描述）
- `variant/template`（变体/模板）
- `routing/policy`（路由策略）

### 2. gateway 层拆出 ModelRouter（审计 §3 合并项②）
新文件或扩展 `backend/app/gateway/model_router.py`：
```
class ModelRouter:
    resolve_provider(profile, role) -> provider
    select_with_policy(task_ctx) -> provider     # 按阶段/风险/预算/隐私/reasoning depth
    fallback_sequence(profile) -> list[provider] # 条件路由，不是死链
    diagnostics() -> health/budget/rate
```
- `gateway/config.py` 继续承担配置解析，**不再承载路由决策**
- `gateway/service.py` 拆成 router + selector + policy evaluator
- `gateway/router.py:132-317` 的 provider 实例化和重试外提进 ModelRouter

### 3. retry/fallback 统一预算 + 错误提取合并（审计 §3 行82，§2.4）
- **不删任何一层**：retry（同 provider）和 fallback（跨 profile）职责各自保留
- 合并 `_status_code_from_exception` 和 `_extract_reason` 成一个公共函数（当前 gateway/router.py 和 fallback_chain.py 各有一份）
- 引入**统一重试预算**：4 层共享一个 `retry_budget`，各层执行前先查剩余预算，避免 `3×N×1` 次失控请求
- `gateway/config.py` 继续承担配置解析，**不再承载路由决策**
- watchdog 配置读取与 gateway 共用 `models.json` 解析层（审计 §2.3 行59，§3 行83）

### 4. 扩 AgentConfig
`models.py:41-63` 新增角色模板引用：
- `planner_model_template`
- `executor_model_template`
- `reviewer_model_template`
- `retrieval_model_template`
- `privacy_level`
- `budget_policy`

### 5. 保持现有 API 不断
- `chat` / `chat_stream` 主链不退化

## 本次不要改什么

- 不要做全量前端模型管理台
- 不要顺手改知识库逻辑（信4 已定型）
- 不要把所有 provider 都大重构一遍
- 不要破坏现有 fallback 可用性

## 精度控制手柄

把这些维度变成显式路由输入：
- 任务阶段（planner/executor/reviewer/retrieval）
- 风险级别
- 预算压力（预算紧张时降模）
- 隐私级别
- reasoning depth（高歧义任务升模）

核心不是“默认模型换成更贵的”，而是“任务/阶段/风险/预算选对模型”。
