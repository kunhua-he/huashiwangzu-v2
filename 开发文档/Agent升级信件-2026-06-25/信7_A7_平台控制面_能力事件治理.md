> 这一封落地审计 §3 合并项③（capability/event 共用 metadata 基类，GovernanceMetadata）。

## 任务目标

本信只解决一个节点：给 capability 和 event 补统一治理 metadata，让它们从“展示页/两张皮”升级成“执行 contract”。
先给 metadata 和治理字段，**不**先做平台全 UI。

## 为什么现在做这个

- 当前已有 capability registry 和 event bus，但治理属性还很轻，且 **capability 和 event 各存各的 metadata（两张皮）**。
- 平台最后不是“能不能调到”，而是“是否可治理、可追踪、可发布、可评估”。
- 这是后续 workflow / scheduler / docs-open / 信13 入库编排复用治理面的底座。

## 当前事实（V2）

- 文件：
  - `backend/app/services/module_registry.py:27-114` capability registry（只存 handler/description/min_role）
  - `backend/app/services/module_events.py:23-80` event bus（解耦广播，无契约分级/重放/死信/耗时诊断）
  - `backend/app/routers/modules.py:18-32` `/capabilities` 返回值（展示用）
  - `modules/agent/backend/models.py:67-230` `AgentEvent` / `AgentHookRun` / `AgentFailureDiagnostic`（可作控制面观测底座）
- 当前缺口：
  - registry 偏轻，缺版本/稳定级别/幂等性/副作用标记
  - 事件与能力两张皮，各存各的 metadata
  - 缺 owner / SLA / activation / quality 等治理属性

## 参考事实（外部更强实现）

- OpenClaw：`README.md:22`（control plane + capability model）
- Coze：`gen_orm_query.go:83-116`（资源和能力一体化）
- Dify：队列/workflow/ops_trace 都进入平台治理面
- 它们强在：capability list 不是展示页，而是执行 contract。

## 本次要改什么

### 1. 抽 GovernanceMetadata 基类（审计 §3 合并项③）
capability 和 event **共用**一个治理 metadata 基类：
```
class GovernanceMetadata:
    owner           # 归属模块
    stability       # 稳定级别
    timeout_ms
    trace_id        # 统一 trace
    quality_score
```

### 2. capability metadata 增强
`module_registry.py:27-114` 新增（不再只存 handler/description/min_role）：
- `owner_module`
- `stability`
- `timeout_ms`
- `rate_limit`
- `trusted_callers`
- `observability_level`
- `quality_score`

### 3. event metadata 增强
`module_events.py:23-80` 新增：
- `delivery_mode`
- `retry_policy`
- `idempotency_key`
- `trace_id`

### 4. /capabilities 升级成 contract
- `routers/modules.py:18-32` 返回值从展示数据扩成可治理 contract
- Agent 调用前能读到治理信息（owner/risk/timeout/contract）

## 本次不要改什么

- 不要先做平台全 UI / 大盘
- 不要改业务模块自己的业务逻辑
- 不要破坏现有跨模块调用边界
- 不要改 runtime（信1 已定型）

## 精度控制手柄

- 不是所有能力都该默认暴露给 agent
- 按可信级别、风险级别、证据需求控制可用面
- 失败原因、超时、重试策略结构化，避免跨模块调用变黑盒

## 验收

### 单测
- 新增 `backend/tests/test_governance_metadata.py`：capability/event 共用基类 / contract 返回
- 现有 `backend/tests/test_module_boundary_contracts.py` 不退化

### 活系统
- `capabilities` 工具返回的 capability 带 owner/risk/timeout/contract
- event 带 trace/retry/delivery metadata
- Agent 调用前能读到治理信息

### 行为口径（做完必须全部成立）
1. capability list 不再只是展示页，而是执行 contract
2. capability 可返回 owner/risk/timeout/contract
3. event 带 trace/retry/delivery metadata
4. capability 和 event **共用同一套 GovernanceMetadata 基类**（不是两张皮）
5. 后续 workflow/scheduler/docs-open 能直接复用治理面
