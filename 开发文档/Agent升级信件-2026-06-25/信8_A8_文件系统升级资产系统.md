# 任务信：信8 — A8 文件系统升级为 agent 资产系统

> 第四层（平台化收口）第 2 封。前置：信1（runtime 产物挂点）。与信11（文档 IR）有协同。
> 依据：总图节点 A8（2183-2256）+ 草稿信 5。

## 任务目标

本信只解决一个节点：把文件从“存储对象”升级成“agent 资产”。
先把 agent 输出挂成 artifact，**不**先重做整个文件中心、桌面 UI、底层去重存储。

## 为什么现在做这个

- 当前文件系统已经是合格的“用户文件系统”，但还不是合格的“Agent 资产系统”。
- 对 agent 来说，文件是：working set / evidence source / intermediate artifact / published deliverable / review target。
- 如果不先把框架文件层升级成资产系统，后续知识库、记忆、workspace、publish、review 都会继续各自维持一套轻量文件语义。

## 当前事实（V2）

- 文件：
  - `backend/app/services/file_service.py:125-177` 已有回收与访问控制
  - `backend/app/services/file_service.py:164-177` `check_file_access()` 统一权限边界
  - `backend/app/routers/file_transfer.py:80-103` 上传链路（成功后发 `file.uploaded`）
  - `modules/knowledge/backend/router.py:790-809` knowledge 已通过 `file.uploaded` 订阅入库
  - `modules/agent/backend/handlers/chat.py:510-559`（信1 后在 runtime）：runtime 结束后产物可挂 artifact 记录
- 当前已具备：去重、删除到回收站、分享、`check_file_access()` 统一权限
- 当前缺口：
  - 没有 agent 资产对象：草稿、产物、证据、发布件、关联会话/任务/能力调用
  - 没有 file lineage / provenance / publish state
  - 没有把 knowledge / memory / workspace / publish 链到同一资产视角

## 参考事实（外部更强实现）

- OpenHands：workspace / 文件选择 / 产物输出之间边界清楚，文件是工程 runtime 的一部分
- Letta：open files / working memory / archival memory 分层关系明确
- Coze / Dify：文件是资源网络的一环，可被 workflow / agent / plugin 稳定引用

## 本次要改什么

### 1. 补一层 agent asset 语义
在框架文件层之上补 artifact 类型：
- `draft`（草稿）
- `published`（已发布）
- `evidence`（证据）
- `generated`（生成产物）
- `handoff`（交付件）

### 2. 建最小资产元数据面
至少能关联：
- `file_id`
- `owner_id`
- `source_session`（conversation_id）/ `source_task` / `source_capability`（tool_name）
- `provenance`（source_file_ids）
- `publish_state`
- `published_by`
- `related_knowledge_doc_id`（如有）

### 3. agent 产物挂 artifact
- runtime（信1 的 task_sink）结束后，把 agent 输出挂成 artifact 记录

### 4. 统一资产语义
- 让 Agent 侧和 knowledge 侧消费**同一**资产语义，而不是各自解释文件
- 为上传 / 生成 / 发布 / 引用补最小事件链或审计链（复用信7 的治理 metadata）

## 本次不要改什么

- 不要顺手重做整个桌面文件 UI
- 不要重做底层内容去重和物理存储
- 不要在本轮引入复杂版本控制系统
- 不要破坏现有 owner/share 权限规则
- 权限控制仍必须经 `check_file_access`

## 精度控制手柄

让这些字段或语义显式可见：
- `asset_type`
- `provenance_type`
- `source_run_id`
- `publish_state`
- `evidence_strength`
- `reusable`
- `garbage_collectable`

重点是让系统能区分：用户原始文件 / agent 中间产物 / 已交付产物 / 检索证据文件。
不是所有输出都自动发布——高风险输出必须先进 draft。
