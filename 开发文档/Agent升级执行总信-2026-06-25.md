## A1 运行时主循环重构

### 当前 V2 逻辑

- `modules/agent/backend/handlers/chat.py:39-99`
  - `_yield_final_stream()`: 流式正文 + inline tool call 兜底
- `modules/agent/backend/handlers/chat.py:101-571`
  - `handle_chat()` / `event_stream()`: 用户消息持久化、工具循环、timeline、hook 收尾
- `modules/agent/backend/engine/engine.py:32-188`
  - `assemble_context()`: prompt / budget / skills / memory / experience 注入
- `modules/agent/backend/engine/engine.py:234-317`
  - `_build_system_content()` + `get_orchestrator()` + `get_hooks()` + `get_budget_tracker()`

当前主链路是：

1. `handle_chat()` 持久化用户消息
2. `assemble_context()` 注入 system prompt / skill / memory / experience
3. `event_stream()` 跑工具循环
4. `_yield_final_stream()` 跑流式正文与 inline tool recovery
5. finally 里补 assistant 持久化、timeline 持久化、post-turn hooks

当前问题：

- runtime 编排过于集中在 `chat.py`
- handler、runtime、stream、task sink 没分层
- 后续 review fork、understanding loop、planner role 没有自然挂点
- stuck / diminishing / budget / stream recovery 都是“存在”，但还不是统一 runtime policy

### 参考后逻辑

- Claude：
  - `src/tools/AgentTool/built-in/planAgent.ts:14-91`
  - `src/tasks/LocalMainSessionTask.ts:1-110`
- LangGraph：
  - `libs/langgraph/langgraph/pregel/main.py`
  - `libs/langgraph/langgraph/pregel/_loop.py`
- OpenHands：
  - conversation runtime / workspace 生命周期闭环

最强逻辑判断：

- Claude 在角色治理上更强
- LangGraph 在可恢复执行和 durability 上更强
- OpenHands 在工程化 runtime 边界上更强

### 谁最强，为什么

- Claude 最强的是角色治理和主会话任务化，适合拿来定义 runtime 骨架
- LangGraph 最强的是显式状态推进和 durability，适合拿来定义可恢复执行 contract
- OpenHands 最强的是 conversation runtime 和 workspace/product boundary 的绑定

### 这一步应该怎么改

先拆 4 层，不先追求完美：

1. `ConversationRuntime`
2. `ToolLoopRuntime`
3. `StreamEmitter`
4. `RuntimeTaskSink`

第一刀只做外提，不顺手塞理解环。

### 必改文件

- `modules/agent/backend/handlers/chat.py`
- `modules/agent/backend/engine/engine.py`
- 新增 `modules/agent/backend/runtime/conversation_runtime.py`
- 新增 `modules/agent/backend/runtime/tool_loop_runtime.py`
- 新增 `modules/agent/backend/runtime/stream_emitter.py`

### 提精度手柄

- `MAX_TOOL_ROUNDS`
- stuck detector
- diminishing returns
- budget pressure
- memory recall quality
- stream 后 inline tool recovery 开关
- final summary fallback 是否启用

这些手柄现在是散在逻辑里，升级后要收口成 runtime policy。

### 先改哪几段函数

1. `modules/agent/backend/handlers/chat.py:146-569`
   - 先把 `event_stream()` 拆出去

   - 把 `_yield_final_stream()` 改成可复用 emitter
