# 知识库底座升级 — 数据清洗 + 管线新增节点方案

## 一、当前数据质量诊断

| 维度 | 数量 | 问题 |
|------|------|------|
| 文档 | 15,628 | 正常 |
| 实体（kb_terms） | 677,625 | **全部是 candidate 状态**，0 confirmed |
| 事实（kb_fact_candidates） | 59,304 | 最高置信度 0.7，20% 低于 0.5 |
| 分块（kb_chunks） | 556,806 | **18% (101,141) 不到50字** |
| 低价值画像 | ~500 | "仅包含一页图片技术分析" 占位描述 |
| 实体前20热词 | RGB/亮度/密度/JPEG | **图片元数据噪声当成了业务实体** |
| 别名（kb_entity_aliases） | 2,821 | 相对 67万实体，合并率极低 |

### 核心病灶

1. **graph 阶段没有主体/标签概念** — 抽出来的实体没有分类（人/产品/机构/成分），全部是 candidate，无法区分噪声
2. **VLM 技术输出污染图谱** — VLM 输出的图片分析信息（RGB、像素、格式等）被 graph 阶段当成实体抽取
3. **profile 阶段与 graph 阶段脱钩** — profile 生成了 subject/doc_type 但没回写到实体图谱
4. **无交叉验证** — 多文档提及同一 fact 时没有互相佐证/升权机制

## 二、方案选择：新增节点（而非重构）

在现有管线后面追加 2 个新阶段，不动已有节点：

```
现有管线:
source_validate → parse_index → page_render → raw_text → raw_ocr → raw_vision
→ fusion → profile → cognitive_index → graph → relations

新增:
→ entity_governance (实体治理)
→ cross_validate (交叉验证)
```

### 为什么新增而不是重构

- 现有节点已跑完 26.5万次，结果没错只是不够精
- 重跑 graph 阶段需要大量 LLM 额度，新节点只做轻量后处理
- 新节点可以反复重跑，不影响原始数据

## 三、新增节点设计

### 节点 1：entity_governance（实体治理）

**目标**：把 67万 candidate 实体分类、去噪、合并，产出 confirmed 主体列表

**处理步骤**：
1. 规则层（零 LLM 成本）：
   - 黑名单正则过滤图片元数据实体（RGB/像素/亮度/密度/格式/尺寸等）
   - 过滤纯数字、单字符、连续标点
   - 过滤 VLM 特征词（photo_like/edge_density/color_histogram 等）
   - 过滤测试数据（smoke/e2e/hello）
   - → 直接标记为 rejected

2. 频率聚类：
   - 出现 >5 次的实体（156万 occurrence 中的高频词）优先处理
   - 出现 =1 次的"孤实体"暂不处理，不降权但标记 low_evidence

3. LLM 分类（批量，用 deepseek）：
   - 输入：实体名 + 出现频率 + 出现上下文（取3条 occurrence 原文）
   - 输出：{ 类型: 人名/产品/机构/成分/证书/术语/噪声, 置信度, 标准化名称 }
   - 按类型写 status=confirmed 并打 entity_type 标签

4. 同义合并：
   - 相同 normalized 名的自动合并
   - LLM 判断相似名是否同一实体（华世王镞/HUA SHI WANG ZU/华世王旌）
   - 合并后保留 alias，occurrence 指向主实体

**数据库变更**：
- kb_terms 新增字段：`entity_type VARCHAR(50)` — 人名/产品/机构/成分/证书/术语
- kb_terms 新增字段：`governance_status VARCHAR(20)` — unprocessed/rule_filtered/llm_classified/merged
- kb_terms 新增字段：`governance_at TIMESTAMP`

### 节点 2：cross_validate（交叉验证）

**目标**：对 fact 做多文档佐证，对画像做质量复核

**处理步骤**：
1. 事实聚合：
   - 同一 subject 的 fact 按 predicate 分组
   - 多文档佐证的 fact：置信度上浮（每多一个独立来源 +0.1，上限 0.95）
   - 孤证 fact：置信度保持或微降

2. 矛盾检测：
   - 同一 subject + predicate 但 object_value 不同 → 标记 contradiction
   - 输出待人工审核队列

3. 画像质量复核：
   - subject 为"仅包含一页/一张…技术分析"的画像 → 重新用 VLM 看原图
   - 低信息画像如果原文档有实质内容 → 重写 profile

4. 置信度重算：
   - 综合来源数、出现频率、实体关联度重新计算 fact 置信度
   - 结果写回 kb_fact_candidates.confidence

**数据库变更**：
- kb_fact_candidates 新增：`corroboration_count INT DEFAULT 0` — 佐证来源数
- kb_fact_candidates 新增：`contradiction_flag BOOLEAN DEFAULT false`
- kb_document_profiles 新增：`quality_score FLOAT` — 画像质量分 0-1

## 四、数据清洗规则（完整）

### 实体黑名单（正则层）

```python
ENTITY_NOISE_PATTERNS = [
    # 图片元数据
    r'^(RGB|RGBA|CMYK|HSL|HSV|sRGB)$',
    r'^(JPEG|PNG|GIF|BMP|WEBP|MPO|HEIC|SVG|TIFF)$',
    r'^\d+x\d+px$',                    # 尺寸 3448x2896px
    r'^x\d+px$',                        # x3448px
    r'^(photo_like|text_like|chart_like|mixed)$',
    r'^(平均亮度|边缘密度|占比|主色|视觉轮廓)$',
    r'^模式$',
    r'^格式$',
    r'^尺寸$',
    r'^其他$',
    
    # VLM 特征标签
    r'^(color_histogram|edge_density|brightness|contrast)$',
    r'^视觉轮廓为$',
    r'^本地图片分析$',
    
    # 品牌拼音碎片（单独出现无意义）
    r'^(HUA|SHI|WANG|ZU)$',
    
    # 通用噪声
    r'^\d+$',                           # 纯数字
    r'^.$',                             # 单字符
    r'^[^\w一-鿿]+$',          # 纯标点
    r'^(smoke|test|e2e|hello)',         # 测试数据
    r'^Artifact Desktop',               # 系统占位
]
```

### 短块清理规则

- <20字 且不含实体引用 → 标记 `quality=junk`，搜索排除
- 20-50字 → 标记 `quality=low`，搜索降权 0.3x
- >50字 → 正常

### 低价值文档标记规则

文档画像 subject 匹配以下模式 → 标记 `low_value=true`：
- "仅包含一页/一张.*技术分析信息"
- "未提供可识别.*正文"
- "低信息量.*记录"
- "烟雾测试"
- "E2E测试"
- "简单问候"

## 五、执行计划

### 第一批（纯规则，零 LLM）
1. 跑正则黑名单，把噪声实体标记 rejected → 预计干掉 10-15万
2. 给短块打 quality 标签
3. 给低价值文档打标签
4. 搜索时加权：rejected 实体不参与、低质量块降权

### 第二批（LLM 分类，用 deepseek）
1. 剩余高频实体（出现>5次）批量分类 → 约 5-10万条
2. 同义合并（华世王镞/HUA SHI WANG ZU 等）
3. 有 confirmed 实体后，搜索按实体类型过滤

### 第三批（LLM 精炼，用 GPT-5.5）
1. 事实交叉验证 — 多文档佐证升权
2. 低价值画像重写 — VLM 二刷
3. 矛盾检测 — 输出人工审核队列

### 自动化触发器（常驻）
- 新文档入库完成 relations 后 → 自动触发 entity_governance
- 每周全量跑一次 cross_validate
- 搜索时实时利用 governance 结果加权
