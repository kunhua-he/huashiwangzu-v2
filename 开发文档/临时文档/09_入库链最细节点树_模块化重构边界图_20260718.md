## 五、污染数据评估探针方案(删库前必经,不一棍子打死)

**华哥铁律:先评估哪些是真污染,别着急动数据库。抽真实数据打探针,才知道怎么打分、跑哪些真污染。**

### 探针分三层,逐层缩小打击面

```
第1层 全局体检(只读SQL,不改数据)
  ├─ 各表行数 + 各状态占比(done/degraded/failed/merged)
  ├─ kb_entity_dictionary:type_id空占比、canonical_id用到占比、长名(>10字)占比
  ├─ kb_graph_edges:各relation占比、相似边(相关)占比、权重分布
  └─ kb_raw_data:各round覆盖、有文本层vs无文本层文档数
  产出:全库污染画像(哪类脏、脏多少)

第2层 抽样打分(抽真实数据,人工+规则判)
  ├─ 随机抽 30 个实体名 → 判:主体名 vs 整句误抽(修饰语没剥)
  ├─ 随机抽 30 条相似边 → 判:真关系 vs 垃圾相似(阈值太松)
  ├─ 抽"华世王镞/娇薇诗"的所有碎节点 → 判:该归并多少
  └─ 抽 10 篇有文本层的文档 → 对比OCR/VLM错字 vs 文本层 → 量化③能纠多少
  产出:每类污染的"真污染率",定阈值

第3层 精准清洗清单(基于打分,只删真污染)
  ├─ 整句误抽实体 → 重抽(不是全删)
  ├─ 垃圾相似边 → 按阈值删(不是全删relation表)
  ├─ 碎节点 → 归并(不是删)
  └─ 确认无价值的死数据 → 才删
  产出:精准SQL清单,每条注明"删什么、为什么是污染、影响几行"
```

### 关键原则
- **能纠不删、能并不删、能重抽不清空**。删是最后手段。
- 每个探针都是**只读优先**,改数据的SQL先出清单给华哥拍,不自动执行。
- 探针脚本落 `临时文档/污染评估探针_日期/`,一个探针一个脚本,可独立重跑。
