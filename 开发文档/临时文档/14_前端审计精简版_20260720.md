## 1. 现状一句话

前端已经是 **可运行的桌面 OS 壳**，不是后台管理页：

| 已立住 | 还在痛 |
|--------|--------|

| Content Open 打开文件 | ElMessage 与桌面 feedback 混用 |

| 双皮肤底座（Win 仅槽） | 无 ESLint / 无单元测试 |
| Playwright 壳层 E2E | `openFileByRecord` 异步 fire-and-forget |

## 3. 方案（按刀施工，别并行开战场）

```text
第1刀 反馈收口（1–2天）
  │
  v
第2刀 拆访达列表巨石（2–4天）
  │
  v
第3刀 openFile Promise化（1天）
  │
  v
第4刀 工程卫生（穿插）
  │
  v
以后再说：完整 Win11 / 全量 modules 迁 products
```

### 第 1 刀：反馈收口（优先，性价比最高）

**目标**：桌面内只剩一套反馈通道。

1. `shared/api/index.ts` 拦截器 **只抛错，不弹 ElMessage**
2. `use-file-operations` / `fm-file-operations` 改 `useAppFeedback`
3. CI / 约定：业务路径禁止直接 `ElMessage*`（login 可白名单）

**验收**：桌面操作不再冒 Element 气泡；失败走 desktop toast/dialog。

---

### 第 2 刀：拆访达列表巨石

**目标**：`fm-file-list.vue` 可改、可测。

1. 按视图拆：`FmGridView` / `FmListView` / `FmColumnView` / `FmGalleryView`
2. selection / dnd / rename 抽公共 hooks
3. 访达 `index.vue` 只留编排，keydown/菜单/prefs 下沉

**验收**：单文件 <400 行量级；补 2–3 个视图切换 Playwright。

---

### 第 3 刀：打开链路 Promise 化

**目标**：打开文件可 await、可测、错误可上报给调用方。

1. `openFileByRecord` 从 `void async + return null` 改为真正 `Promise`
2. handle v2 / 访达 / 桌面图标统一 await
3. 覆盖：resolved / 无产品 / 网络失败

**验收**：调用方拿得到 windowId 或明确失败。

---

### 第 4 刀：工程卫生（穿插做）

1. 上 ESLint（含 modules 禁止 import desktop 内部）
2. 中文文件名英文化  
   - `低内存.ts` / `桌面资源管理器.ts` / `资源缓存.ts` / `应用菜单注册表.ts`
3. `package.json` 的 `name: "-"` / `version: 0.0.0` 正规化
4. 合并 `desktop-shell.css` 与 `desktop-shell-overlay.css` 重复块

---

### 明确先不做

- 完整 Win11 chrome（插槽够用）
- 全量 modules → products 搬迁（先稳契约）
- 大重构 windowManager 状态模型（等前三刀稳住）
