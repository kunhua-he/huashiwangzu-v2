## 未收口 / 契约漂移

### P0 — 必须修

#### 1. `finish_task` 仍串行 `run_test`，与 SOP 不一致

- **位置**：`dev_toolkit/server.py` → `_finish_task`
- **现状**：
  - lint：对 `_split_path_list(lint_paths)` **逐条** `code_lint`（未走多路径并行 `lint()`）
  - 测试：`code_run_test(..., test_targets)` → **单 pytest 进程串行**
- **期望**：与 SOP 一致 → `lint(paths)` 一次并行 + `run_tests_parallel(targets, mode=auto)`
- **影响**：Agent 按 SOP 验完后，`finish_task` 再带 targets 会走慢路径、行为不一致

#### 2. `finish_task` 入参 schema 仍是 string-only

- **位置**：`dev_toolkit/core_tools.py` → `finish_task` inputSchema
- **现状**：
  - `lint_paths`: `type: string`（描述写了 JSON list，schema 非 anyOf）
  - `test_targets`: `type: string`（描述仅「空格分隔」）
- **对比**：`lint` / `run_tests_parallel` / `select_verify` 已是 `anyOf: string | string[]`
- **期望**：schema + 处理层统一 anyOf；`_finish_task` 类型注解允许 list

#### 3. `finish_task` 不自动 `select_verify`（可选增强）

- 收工只吃调用方传入路径/目标，不会 `from_git` 推断
- **建议**：可选参数 `auto_select_verify` / `from_git`；空 target 时补推断（本轮可先不做，P1）

### P1 — 应修

#### 4. `tool_job` lint 强制 `str(path)` 再 split

- **位置**：`dev_toolkit/tool_job_tools.py` → `_build_command` lint 分支
- **现状**：`path = str(arguments.get("path") or "")` 再 `split_path_list`
- **对比**：`run_tests_parallel` job 对 list 已 `json.dumps`；直连 `lint` 直接传 list
- **期望**：`split_path_list(arguments.get("path"))`，禁止先 `str(list)`

### P2 — 文档 / 配置

#### 5. README 只写 Codex `.mcp.json`

- **位置**：`dev_toolkit/README.md` 文首
- **现状**：未写 ZCode 主路径 `.zcode/config.json` → `mcp.servers.project_toolkit`
- **本机事实**：
  - Codex：`.mcp.json` 相对路径 + `PYTHONPATH=.`
  - ZCode：`.zcode/config.json` **绝对路径** + `timeoutMs: 130000`
  - 用户级 `~/.zcode/cli/config.json` 仅网关 MCP（正确）
- **期望**：双入口说明 + finish 与并行 SOP 关系一句

#### 6. `_split_path_list` 注解与实现漂移

- `server.py` 包装注解写 `raw: str`，底层已是 `code_tools.split_path_list`（支持 list）
- 小清理即可

#### 7. CodeGraph CLI 硬编码

- `_CODEGRAPH_CLI = ~/.npm-global/bin/codegraph`
- 无 PATH/`which` 回退；换安装前缀会挂 code 工具
