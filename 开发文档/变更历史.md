## Agent 模块（M0–M2 + 深度强化，2026-06-18 ~ 06-19）

从 V1 PHP 用 V2 架构完整重建：对话 + SSE 流式 + 消息入库、工具框架（技能发现器，读框架能力清单自动生成工具、走 `call_capability`）、三层提示词（系统/企业/个人画像，DB 存储 + 画像自动进化）、桌面感知、终端工具、UI 优化。
- **坑·流式收尾卡死**：`event_stream` 的 `yield [DONE]` 必须放在 `finally` 里，持久化/画像进化用独立 try 包住，否则收尾出错就发不出 `[DONE]` → 前端永久卡"停止"。前端 `done=true`(EOF) 也要 flush streamingText + 加 30s 空闲超时兜底。
- **坑·401**：前端 token key 统一 `v2_auth_token`；Agent 自己的 fetch 要加 401 自愈（清 token 跳登录），否则 token 过期卡死。后端鉴权正常，401 多为前端没带 token。
- **坑·端口**：主框架前端 `127.0.0.1:5173`、后端 `30004`（watchdog 守护）；agent 沙盒前端 `5180`（独立开发壳，有自己的登录态，非主框架）。两者别混。

## 后端稳定性修复（2026-06-18 ~ 06-20）

- **后端暴毙**根因：APP_DEBUG=True 致 SQLAlchemy echo 日志刷爆磁盘 + 无自动重启。修复：APP_DEBUG=False + `scripts/backend_watchdog.sh`（崩溃自动重启 + 日志超 50MB 归档）。
- **watchdog 抢端口**：多个 watchdog 实例互相杀抢 30004（曾 368 次 bind 失败 / 31 次重启）。修复：watchdog 加单实例 PID 锁。
