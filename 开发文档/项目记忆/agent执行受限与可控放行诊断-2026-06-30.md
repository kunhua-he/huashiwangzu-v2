只读诊断。何焜华账号为 admin，terminal-tools exec/write/run_python/publish/import 均已按 editor+ 开放，网络在 macOS sandbox profile 中允许，cwd/读写锁在 data/workspaces/{user_id}，成果可用 terminal-tools:publish 上桌面。聊天中的多项“不能”是假限制：Agent 未发现 publish；把 stdout/stderr 1MB 截断误解为文件大小上限；短链先说无法解析，后用 requests 成功；git clone 失败后 wget tarball 成功；多次 Python SyntaxError 属于命令生成问题。真实代码问题：sandbox._check_path_escape 会把 heredoc 脚本中的字符串字面量 /share/video/{id}/ 当越界路径误拦；sandbox profile 未覆盖 Xcode.app，git/clang 可能被 libxcrun 拦；action_policy 敏感名单写 terminal-tools__execute，但真实能力名是 terminal-tools__exec，导致审批策略漏管；action_policy 未复用 tool_orchestrator 已有 ToolMetadata。建议保留硬边界（宿主机/他人数据/危险系统命令/秘密），自动放行 admin/editor 在工作区内的联网、venv/pip、脚本、下载、生成、publish；replace/delete/外发消息/定时任务/全局配置走确认。新增“不能执行”证据门槛与错误分类自动降级；代码任务优先 run_python/write_file，避免 heredoc 引号错误；需要 JS/登录态时另建隔离 browser-tools，Cookie 不暴露给模型。此项跨 Agent + terminal-tools，应独立于 Agent底座维修09发综合维修信。
