# R6 第二阶段收口

日期：2026-07-03  
Agent：codex-conductor-r6

## 本批目标

用户指出上一批成效偏小，且担心多子代理变成“为了做而做”。本批调整为目的驱动：知识库历史债可大批治理、媒体智能从 placeholder 进入本地事实层、UI gate 明确红项、MCP conductor 降低 5 线程集成成本，并单独开算法研究线程回答“为什么这么做”。

## 结果

1. Knowledge lifecycle debt：新增 pipeline-debt 分类选择和 guarded apply 能力，支持 category/categories/category_limits/limit_each/order；`knowledge:apply_pipeline_debt` 默认 dry_run。活系统 dry-run 证明可安全选择 60 条历史生命周期债（doc_missing=19、source_file_missing=21、source_file_deleted=20），但用户要求本批停下重梳理，因此未执行生产 apply。
2. Knowledge parser defects：`parse_and_index_document` 在进入 parsing 前快照 file_id/extension/content_package_id，降低 greenlet/lazy-load 风险；stale parsing lock 超过 30 分钟且无活跃任务时可安全释放重入。
3. Media intelligence：`local_algorithms.placeholder` 改为 `local_algorithms.local_facts`，用 Pillow 读取图片 metadata 和平均强度 fingerprint，用 ffprobe 读取视频 metadata/timeline markers；OCR/object/small-model/VLM 缺失时返回结构化 degraded，不再 fake placeholder success。
4. Dev toolkit conductor：`agent_board_snapshot` 新增 conductor 汇总，包含 lanes/stale_tasks/recent_memory_links/stage_plan，用于后续 5 线程收工。
5. UI gate：global setup 改为 API token storageState，样例文件改为 repo 内 sandbox samples；全量 UI 跑到 17 passed 后在 hello-world 打开流程失败，定位为明确红项，未继续扩修。
6. Algorithm research：独立线程写入 `开发文档/项目记忆/r6算法研究专线-图片视频知识库算法路线与目标树.md`。结论：自研重点是 schema/pipeline/gating/cache/dedupe/eval/feedback；OCR/ASR/解码/镜头切分/检测/VLM/向量索引优先接成熟库或框架网关。
