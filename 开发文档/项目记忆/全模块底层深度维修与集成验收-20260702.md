总控会话按用户要求完成第一阶段底层维修：以本地工作区为准，围绕“不要两套接口/两套流程、跨模块走统一通路、权限和失败语义稳定、自我降级”做系统性修复。

主要修复面：
- 任务队列：/api/tasks 非 admin 只能看/操作自己的任务，submit 收敛为 admin-only；worker 对 skipped/failed 语义更稳定。
- 文件与 docs-open：新增读写分离的 check_file_write_access；docs-open token 增 edit_doc_ids scope；写入不再 owner impersonation；content/resource/package 写链路统一走权限校验。
- Excel：state_key 由全局唯一改为 (owner_id, state_key)，knowledge_{file_id} 与 import/write 统一走文件权限，不再混用 owner-only 分支。
- Agent/Scheduler：scheduler 改为 user:{creator_id}/system:task-worker 调用身份，不再 scheduler:system/admin 假提升；slow tool 失败返回 failed；subagent_runner owner_id 修正。
- Knowledge：新增 pipeline-debt dry-run 分类治理；source missing/deleted 任务走 skipped/completed；入队统一 enqueue_pipeline_task 并做 dedupe/advisory lock。
- IM：im:send capability 与 HTTP 发送共用同一服务函数，校验 conversation membership；系统通知保留 im:notify。
- Memory：experience 增 owner_id/scope；global 写入限制 admin/system；feedback 原子更新；link 去重。
- Browser/Web：browser-tools 复用 validate_safe_url，Playwright route guard 阻断内网，session 绑定 caller；web-tools 手动逐跳 redirect 校验。
- Codemap/工具台：locks.json read-modify-write 加 OS flock；path 规范化；release gate 和 smoke 对历史 failed 债务做 delta 判定，不因历史队列债假红。
- 前端：image-gen 带 Authorization；doc/ppt viewer normalize parser blocks；office-gen runtime 导出 platform.content；UI E2E 报告失败会影响退出码。
