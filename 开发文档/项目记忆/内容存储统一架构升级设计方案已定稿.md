`开发文档/内容存储统一架构升级设计方案.md`

## 核心结论

### 三条铁律

1. **Agent 禁止操作物理文件（权限硬隔离）** — office-gen:xlsx/docx/replace_existing 对 agent-engine 返回拒绝。Agent 只通过 content:*/excel-engine:* 操作 DB。Agent 可以下载文件到工作区（terminal-tools），但不能直接调 POST /api/files/upload。
2. **DB 存"内容本身"，物理文件是"原始素材"和"编译产物"** — DB 是结构化内容的权威来源（文本+排版+样式+资源引用），物理文件只是用户上传的原文件或从 DB 编译的下载产物。
3. **所有 viewer 读 DB，无缓存则先拉起缓存** — 单线逻辑，不搞 fallback。

### 执行步骤

1. **Step 1（权限隔离 + Agent 工具注册）**: office-gen 能力对 agent-engine 拒绝；agent 工具注册 content:*/excel-engine:* 能力
2. **Step 2（viewer 改读 ContentPackage）**: doc-viewer / ppt-viewer 改读 ContentPackage blocks
3. **Step 3（text-editor + download）**: text-editor 读 blocks；/api/files/download/{id} 支持从 DB 编译
4. **Step 4（excel-engine 缓存一致性）**: 物理文件替换后清 workbook 缓存

### 不兼容风险

- 低：已有 ContentPackage 的 viewer 切换读法数据一致
- 低：无 ContentPackage 的文件 viewer 自动拉起 pipeline
- 中：已有 Agent 脚本调 office-gen:xlsx 会报错，需改调 DB 能力
