# 改了什么

根据外部《华世王镞 V2 代码审计报告》逐项复核并就地修复真实问题：

- 合并 gateway 适配器重复代码：`gemma.py` / `qwen.py` / `openai_compat.py` 的通用逻辑收束到 `backend/app/gateway/adapters/base.py`，registry 继续按模型名分发。
- 合并 `OpenCodeProvider` 薄封装：移入 `backend/app/gateway/openai_provider.py`，删除单独 wrapper 文件。
- 删除确认零引用的后端死代码：`typed_dicts.py`、旧 `model_watchdog/gate_pool.py`、旧 `model_watchdog/router.py`、`route_and_send` 等。
- 统一 `models.json` 配置读取：`backend/app/gateway/config.py` 暴露 `get_models_config` / `get_model_type_config` / `get_provider_configs` / `get_watchdog_model_configs` 等公共 helper；`model_services.py`、`model_watchdog/registry.py`、`gateway/service.py`、agent fallback chain 不再直接重复打开 JSON 或 import 私有 `_config`。
- 统一 parser 与文件型模块读文件入口：新增/强化 `backend/app/services/file_reader.py`，集中 `resolve_caller_user_id`、`read_uploaded_file`、文本解码，并让 parser 模块走 `check_file_access` 后再读盘。
- 抽出 prompt fallback helper：`backend/app/services/prompt_helpers.py` 供 douyin-delivery 与 wechat-writer 复用。
- 删除一批确认零引用前端组件/composable/icon/ui wrapper，保留实际在 viewer 模块、权限指令、命令/RPC 底座中使用的文件。
- 修复 lint 暴露的真实运行时风险：`backend/app/main.py` 缺 `logger`、`file_transfer.py` 缺 `io`、`office-gen/generator.py` 图片分支缺 `Cm`、`scheduler/router.py` 使用未定义 `_j`。
- 修复 event bus SQL jsonb 绑定语法，恢复 assets/workflow router 注册，并补齐 file share ORM 字段以匹配 live DB。

# 有意没有删除的审计项

- 没删 workflow 相关文件：当前 router 已注册，且 `tests/test_platform_workflow_ledger.py` 证明为活能力。
- 没删 `gateway/service.py`：Douyin、WeChat、Agent runtime 仍直接依赖它，外部报告的“零 import”已不成立。
- 没删 `viewer-shell`、`use-permission-action`、`command-registry`、`request-response-channel`：实读确认仍有消费者。
- 没做 parser 六合一与 douyin/wechat 大基类重构：这是更大结构调整，当前先完成低风险底座收束与重复逻辑抽取。
