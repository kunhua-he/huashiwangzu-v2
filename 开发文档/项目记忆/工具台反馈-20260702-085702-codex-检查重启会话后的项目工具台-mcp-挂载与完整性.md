# MCP 使用反馈

## 任务

检查重启会话后的项目工具台 MCP 挂载与完整性

## 顺畅度

- 评分：4/5
- 体感：整体顺畅；项目 MCP 服务端完整，核心工具经 tool_search 后都能加载并调用。

## 本次用到的工具

brief, plan_task, worktree_guard, code_explore, probe, tail_log, routes, capabilities, db_schema, sanity_check, tool_usage_stats, memory_recent, mcp_feedback_summary, finish_task, memory_write

## 卡点 / 不顺手的地方

重启后当前会话初始只暴露 6 个项目工具台工具，必须通过 tool_search 搜具体工具名才能加载更多工具；这容易被误判为 MCP 未完善。

## 缺少的工具 / 能力

建议新增一个轻量 mcp_self_check 工具，直接返回服务端 list_tools 数量、名称、当前组件覆盖率和延迟加载提示。

## 升级建议

可以把 README 的重启后排查写成固定小节：先 tool_search 搜 project toolkit，再搜 code_explore/probe/tail_log 等关键词；也可新增组件健康检查统计 server.py 残留工具。

## 建议移除或合并的工具

无。

## 其他备注

本轮未修改产品代码；按项目规则写入了一条任务记忆。
