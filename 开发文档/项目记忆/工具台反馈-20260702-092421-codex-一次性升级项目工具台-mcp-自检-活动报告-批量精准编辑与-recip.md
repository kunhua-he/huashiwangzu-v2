# MCP 使用反馈

## 任务

一次性升级项目工具台 MCP：自检、活动报告、批量精准编辑与 recipe 编辑

## 顺畅度

- 评分：4/5
- 体感：整体顺畅；组件化模式很好用，新工具可以不碰业务代码完成扩展，测试也能快速覆盖。

## 本次用到的工具

plan_task, worktree_guard, code_node, code_impact, finish_task, memory_write, mcp_feedback

## 卡点 / 不顺手的地方

当前 MCP 进程不能热加载新增工具，本轮新工具只能通过本地导入 server.py 自测，需重启会话或重连 MCP 后 tool_search 才能看到。

## 缺少的工具 / 能力

后续可以做一个真正的 MCP reload/self-restart 提示工具，但是否能热重载取决于 Codex 宿主。

## 升级建议

下一步可以继续拆 server.py 中 system/workflow 直写工具，把 server.py 控到 600 行以下；也可以接入 ast-grep 作为可选结构化编辑后端，但默认仍保持 quick_fix 安全文本引擎。

## 建议移除或合并的工具

暂无；新工具分层清楚，quick_fix 单点、batch 多点、recipe 常见模式、insight 做复盘。

## 其他备注

本次新增工具数使本地 server.list_tools 从 43 增至 51。
