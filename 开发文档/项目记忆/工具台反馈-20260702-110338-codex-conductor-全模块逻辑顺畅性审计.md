# MCP 使用反馈

## 任务

全模块逻辑顺畅性审计

## 顺畅度

- 评分：5/5
- 体感：整体顺畅，工具台健康/能力/路由/release gate 对审计很有帮助。

## 本次用到的工具

brief, plan_task, worktree_guard, code_explore, capabilities, routes, tail_log, probe, call_capability, finish_task, memory_write, mcp_feedback

## 卡点 / 不顺手的地方

capabilities 输出很长且不方便直接做 manifest vs runtime 差分；tail_log 默认为空时无法确认是否读到了正确日志。

## 缺少的工具 / 能力

希望增加模块契约审计工具：一次输出 manifest component/router/public_actions/register_capability/min_role/runtime/sandbox/README/TS any 的结构化差分。

## 升级建议

release_gate 已很好，可把 sandbox skip 明细、后台任务 recent failed 样本和 capabilities 差分合并进一个 audit_modules 命令。

## 建议移除或合并的工具

无

## 其他备注

本次配合子代理效果好，但最终仍需要主控复核高危结论。
