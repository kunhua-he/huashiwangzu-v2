# 执行信：前端类型安全与假空态全局清扫

## 任务定位

这封信专门处理前端横向质量问题：类型安全、API 字段对齐、LoadState/ApiError 覆盖、假空态清扫。它可以与后端/模块总攻并发，但注意不要大改业务逻辑。

## 目标

把前端从“失败时假空、类型绕过、字段靠猜”推进到“失败可见、类型可信、字段与后端契约一致”。

## 范围

```text
frontend/src
frontend/tests
modules/*/frontend
```

重点：

```text
frontend/src/shared/api
frontend/src/shared/composables
frontend/src/shared/components/load-state-banner.vue
frontend/src/shared/components/notification-panel.vue
frontend/src/desktop
modules/agent/frontend
modules/knowledge/frontend
```

## 必做

1. 全局搜索并清理：`any`、`as any`、`@ts-ignore`、`@ts-expect-error`。
2. 对未知 API 响应定义接口，不允许用 any 压过去。
3. 检查后端字段名与前端访问字段是否一致，例如 `entry_component_key` 不能读成未定义 camelCase。
4. `转中文()` 只能用于 UI 文案，不能用于字段名转换。
5. 主要数据页必须有 LoadState：idle/loading/ready/error/stale。
6. 失败时不能显示假空态：必须显示 error banner、重试按钮、后端错误信息摘要。
7. 桌面文件、Artifact、Agent、Knowledge 主要入口都要覆盖 ApiErrorInfo。
8. 通知中心错误要可复制、可跳转来源。
9. Playwright 测试补充关键失败态，不硬等。
10. `npm --prefix frontend run build` 必须通过。
