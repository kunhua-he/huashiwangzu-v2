# 执行信：后端语义失败统一与假绿归零

## 目标

统一后端所有“语义失败”判断，消灭 HTTP 200 / 外层 success:true 包住内部 error 的假绿。

## 修改边界

允许：

```text
backend/app/routers/modules.py
backend/app/services/module_registry.py
backend/app/services/task_worker.py
backend/app/services/task_queue_audit_service.py
backend/app/main.py
dev_toolkit/smoke.py
dev_toolkit/release_gate.py
dev_toolkit/release_response.py
backend/tests/
dev_toolkit/test_*.py
开发文档/项目记忆/
```

禁止：

```text
frontend/
modules/agent/
modules/knowledge/
backend/app/services/content/
```

## 必做

1. 新增统一函数，例如 `semantic_failure_reason(result)`，覆盖：
   - `success:false`
   - 顶层/内层 `error`
   - `status in failed/error`
   - legacy `code != 0`
2. `/api/modules/call` 使用统一语义失败判断，不再只看 `success is False`。
3. task worker / task audit / health / smoke / release_gate 复用同一口径或同等规则。
4. `POST /api/tasks/submit` 对无 handler 的 task_type 不得先成功入队；必须拒绝或明确 accepted-but-not-executable。
5. codemap/direct HTTP 与 `/api/modules/call` 的错误语义尽量一致。

