## 3. 必修项

### P0 — release gate/MCP 收敛可靠性

现状：

- `dev_toolkit/release_gate.py` 已有未提交改动：`--skip-ui` 标记 UI coverage debt，`build_release_summary(skip_ui=...)` 区分 `clean_pass/release_safe/has_debt`。
- `dev_toolkit/release_response.py` 已有未提交改动：`PASS_WITH_DEBT` 不再映射为 clean success。
- 但仍有风险：
  - MCP `release_gate(skip_ui=true)` 可能 30 秒超时。
  - server 内 `_release_gate()` 内部等 600 秒，外层可能先断，造成工具悬空。
  - `release_response.py` 目前如果缺少 `RELEASE_GATE_JSON` 且 returncode=0，仍可能 fallback 成 PASS（需 fail closed）。

建议修法：

1. `dev_toolkit/release_response.py`
   - 缺失/无法解析 `RELEASE_GATE_JSON` 时 fail closed：
     - `success=false`
     - `clean_pass=false`
     - `release_safe=false`
     - `verdict="INVALID_GATE_OUTPUT"`
     - `parse_error=true` 或 `invalid_output=true`
     - 保留 `output_tail`。
   - 保持 `PASS_WITH_DEBT`：`success=false`、`clean_pass=false`、`release_safe=true`。

2. `dev_toolkit/release_gate.py`
   - 增加快速 `--preflight` 模式：只跑 health、system status、queue baseline/audit、semantic scan、UI coverage 标记；不跑耗时 smoke 和 sandbox `--check`。
   - full 模式保留现有 smoke + sandbox matrix。
   - `--skip-ui` 永远不能 clean pass，只能 `PASS_WITH_DEBT` 或 blocker。

3. `dev_toolkit/server.py` / `dev_toolkit/core_tools.py`
   - MCP `release_gate` 增加 `mode` 参数：`preflight` 默认，`full` 可选。
   - `_release_gate()` timeout/cancellation 时必须 terminate/kill 子进程，返回结构化失败，不留下 orphan。

4. 测试：
   - `dev_toolkit/test_release_response.py`：缺少机器 JSON fail closed；skip-ui debt；PASS_WITH_DEBT release_safe true。
   - `dev_toolkit/test_release_gate.py`：preflight 不跑 smoke/sandbox；summary 字段正确。
   - `dev_toolkit/test_server_helpers.py`：server 响应构造/timeout 行为。
   - `dev_toolkit/test_mcp_entry.py`：schema 暴露 `mode`。

### P0 — reset_runtime_data.py 必须加固后才可入仓

现状：

- 新脚本：`backend/scripts/maintenance/reset_runtime_data.py`
- 当前有用，但不宜原样提交：
  - `--yes` 太弱。
  - 默认逻辑是 truncate 所有 public base table except keep set，未来新表会被自动清掉。
  - 没有硬性非生产保护。
  - 没有要求 DB 备份证明。
  - runtime 文件清理路径保护不足。
  - 没有测试。

建议修法：

1. CLI：
   - dry-run 默认。
   - apply 必须：`--yes --confirm "RESET <db_name>"`。
   - 增加 `--scope`：`tasks | knowledge | agent | files | all-runtime`。
   - 增加 `--db-backup`：指向已存在 DB 备份文件或包含 `database.sql/manifest.json` 的目录。
   - 如需非本地 DB，必须额外显式参数（例如 `--allow-non-local-db`），默认拒绝。

2. 安全：
   - DB host 默认只允许 `127.0.0.1/localhost/::1`。
   - DB name 含 `prod/production/prd` 等生产词直接拒绝。
   - scope 使用显式表 allowlist，不再“所有非 keep 表”。
   - runtime dir resolve 后必须在 `backend/data/` 下。
   - 拒绝 symlink。
   - 拒绝清理 repo root/backend root/backend/data 本身。
   - `--backup-dir` 不得位于任何即将清理的 runtime dir 内。

3. 输出 JSON：
   - `applied`
   - `scope`
   - `database`
   - `truncate_tables`
   - `available_tables`
   - `skipped_tables`
   - `clean_files`
   - `archived_dirs`
   - `cleared_dirs`
   - `db_backup`
   - `backup_dir`

4. 新增测试：`backend/tests/test_reset_runtime_data.py`
   - 确认字符串缺失/错误拒绝。
   - 生产 DB 名拒绝。
   - 非本地 host 拒绝。
   - scope 表选择只包含显式表。
   - dry-run 不执行 truncate、不清文件。
   - `--clean-files --yes` 要求 `--backup-dir`。
   - backup dir 在 runtime dir 内拒绝。
   - runtime symlink 拒绝。
   - DB backup 缺失拒绝。
