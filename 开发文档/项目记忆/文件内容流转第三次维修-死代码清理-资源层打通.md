## 改了什么

### 死代码清理
1. **删除 `backend/app/routers/office_export.py`** — 废弃的旧 Office 导出路由，自 content:export + office-gen 上线后无人调用。从 `registry.py` 移除注册。
2. **删除 `backend/app/services/content/adapter.py:export_via_office_service()`** — 无调用者的死代码，旧路由直连 Framework Office Services。
3. **删除 Framework Office Services 三文件** — `docx_service.py`, `excel_service.py`, `pptx_service.py`（仅被已删的 office_export.py 使用）。保留 `text_editor_service.py`、`csv_editor_service.py`（活跃使用）。
4. 更新 `backend/app/services/office/__init__.py` 移除已删服务的导出。

### 资源层打通
5. **新增 `content:store_resource` capability** — 接受 base64 编码的二进制数据 + 元数据，通过 `ResourceService.create_resource()` 写入 `framework_resources`（内容寻址、hash 去重）。已在 content router 注册。
6. **pptx-parser 提取内嵌图片** — `python-pptx` 的 `shape.image.blob` 提取二进制，base64 编码后回传给 pipeline。
7. **docx-parser 提取内嵌图片** — `python-docx` 的 `rel.target_part.blob` 提取二进制，base64 编码后回传。
8. **pdf-parser 提取内嵌图片** — `PyMuPDF(fitz)` 从 PDF page 提取 pixmap（png 格式），base64 编码后回传。
9. **`ContentPackageService.run_pipeline()` 直接保存资源** — 解析结果中的 `_bytes_b64` 字段由 pipeline 一站式通过 `ResourceService.create_resource()` 写入，自动替换 blocks 中的本地 resource_ref 为真实 resource_id，并创建 `ResourceRef`。

## 残留风险
- parser 模块的异步 `call_capability("content", "store_resource", ...)` 与 pipeline 的主流程分别存储。依赖 `ResourceService.create_resource()` 的 hash 去重保持幂等（ref_count 会+1，无害）。
- `framework_resources` 存储路径为 `res/{sha256[:2]}/{sha256[2:4]}/{sha256}.{ext}`，与 `framework_file_items` 的 `uploads/{md5[:2]}/{md5[2:4]}/{md5}.{ext}` 不共用存储层。后续任务可考虑统一。
- `excel_workbooks` 编辑态 → Content Package 同步仍未实现。
