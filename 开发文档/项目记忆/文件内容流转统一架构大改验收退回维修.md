## 改了什么

**P0 修复 (4项):**
1. KbDocument 迁移 — 给 kb_documents 补齐 content_package_id 列 (ALTER TABLE + model + init_db migration)
2. Content capability handler owner 权限 — 全部 14 个 capability 统一用 resolve_caller_user_id 获取 owner_id 并传给 service，_cap_store_resource 不再用 params.get("owner_id", 0)
3. office-gen 集成 Content Package — _cap_generate_to_artifact 生成后创建 Content Package (origin_type="generated")，返回 content_package_id；_cap_replace_existing 同步更新 Content Package；_cap_export_to_artifact 优先读已有 Content Package
4. content:export 冲突处理 — 增加 conflict_policy 参数 (auto_rename/create_version/overwrite)，同名不卡死；连续 export 两次验证通过

**P1 修复 (4项):**
5. Agent 工具指引 — DEFAULT_TOOL_GUIDES 新增 content:* 条目排首位，office-gen/excel-engine 降级为底层 adapter；degradation recipe 优先 content:* 工作流
6. null source_file_id — ContentPackage 模型加 origin_type 字段 (uploaded/generated/manual/imported)，get_or_create 和 office-gen 创建时写入。已有 null source_file_id 的 package 保持兼容
7. Knowledge 深度集成 — register_document 找不到 content_package 时自动触发 content:pipeline；parse_and_index_document 递归 flatten blocks children 并保留 block_id；document_payload 返回 content_package_id；KbChunk 加 block_id 字段；search 结果返回 content_package_id + block_id
8. 旧 Office API — /api/office/ 路由桥接到 Content Package 服务（/api/content/）；runtime/index.ts 新增 content 命名空间 + apiPut；旧 OfficePackage 接口扩展 content_package_id 字段

**P2 (1项):**
Lint 修复 — package_service/export_service/office-gen router 全部 I001/F401/E712 修复，9 个文件 lint 全过
