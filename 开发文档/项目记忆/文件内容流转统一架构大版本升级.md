## 改了什么
**新增框架级模型 (4张表):**
- `framework_content_packages` — 逻辑内容唯一真相源
- `framework_content_package_versions` — 内容版本快照
- `framework_resources` — 媒体资源内容寻址存储
- `framework_resource_refs` — 资源与包/block/版本绑定

**新增框架级模式:**
- `backend/app/schemas/content_package.py` — Content IR (manifest + blocks + resources)

**新增框架级服务 (5文件):**
- `backend/app/services/content/package_service.py` — 包CRUD + 编辑 + 版本
- `backend/app/services/content/resource_service.py` — 资源存储/去重/引用
- `backend/app/services/content/pipeline_service.py` — parse + package + resource extraction
- `backend/app/services/content/export_service.py` — Content Package → 物理文件
- `backend/app/services/content/adapter.py` — 格式导出适配器

**新增框架级路由+能力 (14个content:*):**
- `backend/app/routers/content.py` — REST + capability handlers + event handler
- pipeline / get_package / get_full_package / list_blocks / get_block / update_blocks / append_blocks / replace_text / export / publish / list_versions / restore_version / list_resources / get_resource

**修改:**
- `backend/app/models/__init__.py` — 导入新模型
- `backend/app/routers/registry.py` — 注册content路由
- `modules/knowledge/backend/models.py` — KbDocument 增加 content_package_id
- `modules/knowledge/backend/services/document_service.py` — parse_and_index_document 优先读Content Package

## 验证了什么
- 后端健康 + 模块导入正常
- 14个content:*能力全部注册可见
- 上传txt文件 → content pipeline → package_id=2, status=parsed
- get_full_package 返回完整manifest + blocks + resource_refs
- list_blocks 返回blocks列表
- get_block 按block_id查单block
- update_blocks 更新文本 + 自动创建新版本
- list_versions 返回所有版本
- REST /api/content/packages/{id} 正常

## 残留风险 P1
- Framework office services (DocxService等) 仍是旧解析路径, 未合并到parser模块
- excel-engine publish 仍直接生成物理文件, 不走content export
- office-gen 部分能力尚未桥接到Content Package
- framework_file_json_packages 标记deprecated但尚未废弃
- Agent 工具指引尚未创建 content:* 偏好的 worklow recipe
