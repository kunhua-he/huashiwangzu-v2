## 概要
完成了文件从拖入/上传 → 登记 → 解析 → 知识库 ingest → Agent 读取 → 编辑 → 导出下载的完整现状追踪和统一架构设计。

## 关键发现
1. **Office 格式解析有三套平行实现**：Framework Office Services (`backend/app/services/office/`)、Parser 模块 (`modules/*-parser/`)、Excel Engine 自解析。三套代码做同样的事。
2. **没有统一内容包**：`framework_file_json_packages` 内容为空壳，Parser 模块输出格式与 Framework Service 输出格式不同。
3. **没有统一资源存储**：PPTX/DOCX 内嵌图片未提取到独立资源表，同一图片在多个文件中存多份。
4. **Excel 有独立版本体系**脱离框架。
5. **Agent 无法读取结构化内容**——只有 knowledge chunk 文本碎片。

## 核心结论
"文件内容"的唯一真相源分两层：
- **物理层**：`framework_file_items` + `framework_resources`
- **逻辑层**：`framework_content_packages`（含 manifests + blocks + resource_refs）

不同文件类型的 canonical source 可以不同（xlsx = Content Package + excel_workbooks；docx/pptx = Content Package；image = framework_resources），但统一入口是 Content Package。

## 推荐动作
三轮迭代：
1. 创建 framework_resources + framework_content_packages 表，标准化 parser 输出
2. 实现 content:pipeline 自动化框架能力
3. 创建 Agent 统一 content:* 工具层，废弃旧表
