### 仍存在的问题（待维修）

1. ❌ **`framework_resources` 表完全为空** — 0 条记录
   - 根因：所有 parser 模块返回的 `resources` 列表为空（只返回 metadata 不提取内嵌资源）
   - 影响：PPTX/DOCX 内嵌图片没有被提取到独立资源表；同一图片在多个文件中存多份
   - 修复方向：`docx-parser`、`pptx-parser`、`pdf-parser` 需要从 Office XML 中提取内嵌图片/BLOB，写入 `framework_resources`

2. ❌ **`framework_file_json_packages` 和 `framework_file_json_versions` 空表未清理**
   - 数据量：0 条（已经是空壳，可直接考虑废弃）
   - Office-gen 的 `/api/office/package` 端点路由仍然指向 Office JSON Package 而非 Content Package

3. ❌ **Agent 未暴露 content:* 统一工具层**
   - Agent 工具列表中仍为 `desktop-tools:read_file`（读原始物理文件）+ `desktop-tools:create_file`（创建文本文件）
   - `content:list_packages`、`content:get_package`、`content:query_blocks`、`content:update_blocks` 等能力虽然注册了，但 Agent 没有通过 tool_discovery 暴露给用户
   - `tool_discovery.py` 中未注册 `content:*` 工具类

4. ❌ **Framework Office Services 与 parser 模块仍为平行实现**
   - `backend/app/services/office/excel_service.py` 和 `modules/xlsx-parser/backend/router.py` 都自己做 xlsx 解析
   - `backend/app/services/office/pptx_service.py` 和 `modules/pptx-parser/backend/router.py` 都自己做 pptx 解析
   - `backend/app/services/office/docx_service.py` 和 `modules/docx-parser/backend/router.py` 都自己做 docx 解析
   - Content export 时的 `export_via_office_service()` 分支走 Framework Office Service（老路），不走 parser module
   - 修复建议：export 时也应该通过 `call_capability` 调 parser module

5. ❌ **excel_workbooks 编辑态与 content package 无同步机制**
   - `excel-engine:update_range` 操作直接改 `excel_cells` 表
   - `excel-engine:export_xlsx` 直接从 `excel_cells` 编译 xlsx，不走 Content Package
   - Content Package 的 blocks 中没有同步反映 excel_workbooks 改动

6. ❌ **'content:*' 工具未进入 Agent tool discovery**
   - 检查了 `modules/agent/backend/services/tool_discovery.py`，缺少 `content:` 前缀工具注册
   - Agent 用户在聊天界面无法通过自然语言调用 `content:update_blocks` 等

### 核心结论

V2 "文件内容"唯一真相源当前已是两层的：

- **物理层**：`framework_file_items` + 磁盘文件
- **逻辑层**：`framework_content_packages`（含 blocks + version history）

目标需补：
- **资源层**：`framework_resources` + `framework_resource_refs`
- **工具层**：Agent 通过 `content:*` 统一工具操作 Content Package

当前 pipeline 通、schema 对、版本管理有，但资源层（framework_resources 空表）和工具层（Agent 未暴露 content:* 工具）两处核心缺失，xlsx 编辑态同步是第三个大缺口。
