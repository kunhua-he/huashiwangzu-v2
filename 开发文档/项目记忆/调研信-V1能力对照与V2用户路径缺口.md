# 调研信：V1 能力对照与 V2 用户路径缺口

## 任务定位

只读调研信，可与总攻大信并发。目标不是改代码，而是避免 V2 重建过程中漏掉 V1 的关键用户能力。

## 目标

从用户路径角度对照 V1 和 V2，列出 V2 仍缺的关键能力、已替代能力、无需恢复能力、需要重新设计能力。

## 必做

1. 列出 V1 的主要用户功能：文件、知识库、AI/Agent、内容生成、解析、桌面/工作台、权限、导出。
2. 对照 V2 当前实现，按能力给出状态：已完成 / 部分完成 / 缺失 / 不应恢复 / 需重设计。
3. 标出 P0 用户路径缺口：用户今天会明显卡住的地方。
4. 标出 P1 产品增强：能用但体验不完整。
5. 标出 V1 中不该带回来的旧包袱。
6. 标出哪些缺口已经被“全链路产品化落地总攻大信”覆盖，哪些没有覆盖。
7. 给出下一批大信建议，而不是小碎片建议。

## 输出

写报告到：

```text

```

```text
V1能力 | V2状态 | 用户影响 | 是否总攻覆盖 | 建议动作 | 优先级
```
