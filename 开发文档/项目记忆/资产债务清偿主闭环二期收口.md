## 本轮修正

- 文件永久删除链路：`delete_permanently` / `empty_trash` 返回 `permanently_deleted_file_ids`，router 按 ids 发 `file.permanent_deleted`；文件夹递归永久删除和清空回收站覆盖子文件；递归删除补 owner 过滤，异常跨 owner 子行不删不发事件。
- 文件软删除链路：文件夹软删除/恢复对子文件发 `file.deleted` / `file.restored`；desktop-tools 删除路径改为创建 recycle row 并发 `file.deleted`。
- ContentPackage：新增 `content:archive_lifecycle_unavailable_packages` 和 `content:repair_missing_current_versions`；真实执行必须 confirm；归档只改状态/parse_error/manifest lifecycle，不删 package/version/resource；已生命周期归档包不再被重复改写 provenance，`audit_reason` 写入 manifest。
- Knowledge：`archive_source_unavailable_documents` 保持只归档 deleted/missing 源文件；capability reason 增加白名单校验。
- 测试污染工具：强 marker 收紧为 `smoke- / e2e- / recycle- / pytest- / test-upload- / test-file- / test-pollution- / lifecycle-source- / permanent-source-`，移除裸 `test-`；cleanup 可归档 filename-only orphan Knowledge 文档；物理删除根目录按后端 `UPLOAD_DIR` 解析并限制在 upload root 内。
- Release gate：ContentPackage lifecycle blocker 改看未归档源不可用与真实 missing-current blocker；preflight/skip-ui 保持 `clean_release_ready=false`，`PASS_WITH_DEBT` 可部署但不算 clean release。

## 残留风险

- 永久删除事件仍是删除事务 commit 后由 router best-effort 发出；如果进程在 commit 后、事件持久化前崩溃，理论上仍可能漏 outbox 级保证。当前按执行信建议保持 service 不硬 import event bus。
- 当前验收是 preflight/skip-ui，不是 full UI clean release；因此 `clean_release_ready=false` 是预期语义。
