## 代码收口

- ContentPackage lifecycle service 增加统一状态：`source_available/source_recycled/source_missing/source_permanently_deleted/archived_by_lifecycle/restored`，并支持删除、恢复、永久删除事件的幂等归档/恢复。
- `/api/recycle` 永久删除和清空回收站会对所有涉及的文件 id 发 `file.permanent_deleted`，包括文件夹子文件。
- `/api/files/delete`、批量删除和 `/api/recycle/restore` 对文件夹递归发子文件 `file.deleted/file.restored`。
- `desktop-tools:delete_file` 不再绕过生命周期：写回收站记录并发 `file.deleted`。
- Knowledge lifecycle audit 计数不再被 `limit` 截断，dashboard 不再把 source unavailable 计入 failed。
- Test pollution marker 包含执行信要求的 `test-`，并保留更精确 marker；真实清理物理上传文件前会检查非候选文件的 `storage_path/md5_hash` 引用，避免误删共享物理文件。
- release gate 三个资产项均输出机器可读 data，并包含 `clean_release_ready` 与 `deploy_allowed`。
